DESARROLLO DE SITIOS WEB CON TRANSACCIONES EN LÍNEA

Es preciso identificar primeramente lo referente al comercio, se trata de llevar a cabo transacciones con la finalidad de comprar o vender. Ahora bien, comercio electrónico, hace referencia a la actividad involucrada en el intercambio de bienes o servicios, lo anterior apoyado en el uso de las Tecnologías de la Información y las Comunicaciones (TIC) como plataforma principal para su promoción y distribución.

Comercio electrónico

1.-Definición

La posibilidad de replicar procesos de venta por medios electrónicos, principalmente vía internet, provoca el nacimiento de una nueva actividad empresarial denominada comercio electrónico ya en los años 90 del siglo pasado. (Somalo, 2017, p.15).

2.-Definición

…el comercio electrónico se define como la venta o compra de bienes o servicios que se realiza a través de redes informáticas con métodos específicamente diseñados para recibir o colocar pedidos, de acuerdo con la definición de la OCDE establecida en 2001 y revisada en 2009. (OCDE, 2020, p. 17).

3.-Definición

Un concepto general, que abarca cualquier forma de transacción comercial o intercambio de información llevado a cabo usando tecnologías de información y comunicación (TIC), el comercio electrónico tiene cabida entre compañías y las administraciones públicas. El comercio electrónico incluye la comercialización electrónica de bienes, servicios y material electrónico. (Whiteley, 2000, p. 59 citado por Robles Estrada & Ortiz Barrera).







Lenguajes Orientados a Web

***Lenguajes de programación más utilizados en la actualidad para el desarrollo de sitios web con transacciones en línea

JavaScript. Es el único lenguaje de programación que los navegadores entienden. Se desarrolla la parte de funcionalidad frontened en sitios web y aplicaciones web modernas. Sus usos más importantes son en: Desarrollo de sitios Web del lado del cliente, desarrollo de todo tipo de aplicaciones por su plataforma NodeJS, desarrollo de aplicaciones de escritorio para SO Windows, Linux y Mac.

Asychronus JavaScript and XML (AJAX). Se refiere a un grupo de tecnologías utilizadas para desarrollar aplicaciones Web.Permite que un usuario de la aplicación web interactúe con una página web sin la interrupción que implica volver a cargarla. Se compone de las siguientes tecnologías: XHTML, DOM, El objeto XMLHttpRequest, XML, HTML y XSLT, Se visualiza JavaScript para enlazar solicitudes e información de datos. Sus limitaciones: No todos los navegadores admiten JavaScript o el objeto XMLHttpRequest, tomar en cuenta la privacidad de usuario a la hora del desarrollo de la aplicación, podría ser que el historial del navegador y los marcadores no se comporten correctamente, no es posible realizar búsquedas en aplicaciones AJAX.

JQuery. Facilita el desarrollo de aplicaciones enriquecidas del lado del cliente, en JavaScript, compatibles con todos los navegadores. JQuery no es un lenguaje, sino una serie de funciones y métodos de JavaScript. JavaScript ← Lenguaje JQuery ← Librería A veces se le refiere a JQuery como framework o incluso un API de funciones. La principal ventaja ante la llegada de JQuery: Se deja la preocupación sobre qué navegador utilizar porque ejecutará el código que sea compatible con el software del cliente que está accediendo a la web. JQuery proporciona: Un grande abanico de funcionalidades que se extiende por medio de miles de plugins.

PHP. Lenguaje de código abierto muy popular especialmente adecuado para el desarrollo de aplicaciones Web. Puede ser incrustado en HTML, le diferencia de JavaScript que el código es ejecutado en el servidor, generando HTML y enviándolo al cliente.

JSP. Se trata de una tecnología orientada a crear páginas web con programación en Java. Es posible crear aplicaciones web que se ejecuten en variados servidores web, de plataformas múltiples derivado a que Java es un lenguaje multiplataforma. Las JSP se pueden escribir en nuestro editor HTML/XML habitual.

ASP.Net. .Net es una plataforma de desarrollo compuesta por herramientas, lenguajes de programación y bibliotecas para crear aplicaciones. ASP.Net ← Se trata de la ampliación de .Net con herramientas y bibliotecas encaminadas a la creación de aplicaciones Web Los sitios web se crean mediante: • HTML • CSS • JavaScript

Ruby On Rails. Es un framework o marco de creación de aplicaciones de código abierto que funciona con Ruby. Rails al igual que Ruby, está orientado a objetos y utiliza su dinamismo para facilitar el desarrollo de aplicaciones web utilizando Ruby. Es fácil de leer y ofrece gran ventaja para cualquier programador sin experiencia. Se basa en dos principios. • No te repitas • Convención sobre configuración. En donde la primera advierte a los desarrolladores que escribir el mismo código reiteradamente podría ser contraproducente y la segunda, indica que Rails supone lo que quiere hacer y cómo, entonces, lo hace sin la necesidad de especificaciones detalladas. Fue concebido para desarrollar aplicaciones web para empresas emergentes, utilizando la metodología de desarrollo ágil de software. Debido a la facilidad que posee se visualizan resultados rápidamente.

***Hablemos acerca de las siguientes tecnologías:

Maquetación o HTML o CSS.- La maquetación se realiza utilizando el lenguaje HTML para la presentación de información. Se utiliza la etiqueta:

El estilo en las divisiones se realiza a través de CSS.

Intercambio de datos o XML o JSON.- Los programadores, desarrolladores y profesionales de TI requieren de la capacidad de transferir estructuras de datos desde cualquier lenguaje a formatos reconocibles por otros lenguajes y plataformas. JavaScript Object Notation (JSON) – es el formato de cambio de datos que lo hace posible. • Legible por humanos • Ligero • Requiere menos codificación • Procesa más rápido. XML es un lenguaje pensado para el intercambio de información. Se trata de un lenguaje de etiquetas mediante el cual es posible modelizar cualquier tipo de datos tan complejo como necesario sea. Ventajas: • Su formato es estructurado y fácil de comprender por la máquina. • Su uso facilita la definición de estructuras complejas que pueden reutilizarse. Desventajas: • Difícil de trabajar por las personas. • Tiempo prolongado de procesamiento. • Un error con el namespace invalida todo el documento. Diferencias entre JSON y XML. XML se aplica principalmente a la comunicación entre servidores y aplicaciones ya que tiene la capacidad de crear mensajes más complejos que sin problema pueden ser validados. JSON es un formato sencillo por lo que es la solución idónea para dispositivos que no requieren de grandes procesos.

Programación o AJAX o JSP o SERVLETS o ASP o NODEJS.- La interacción de un sitio web depende de la utilización de un lenguaje de programación. AJAX (Asychronus JavaScript and XML). Se refiere a un grupo de tecnologías utilizadas para desarrollar aplicaciones Web. JSP. Se trata de una tecnología orientada a crear páginas web con programación en Java. Los servlets son utilizados para ejecutar aplicaciones alojadas por servidores web. ASP. Es un lenguaje interpretado, su código está embebido en HTML. Node.js permite trabajar con JavaScript del lado del servidor. Gestiona todas sus tareas de una forma asíncrona.

Bibliotecas 5 6 o JQUERY o BOOTSTRAP.- JQuery es un framework de JavaScript que permite cambiar el contenido de una página web sin la necesidad de recargarla. • Es más fácil realizar aplicaciones complejas del lado del cliente. Bootstrap es un framework CSS utilizado en aplicaciones front-end (pantalla de interfaz con el usuario). Se utiliza para desarrollar aplicaciones que se adaptan a cualquier dispositivo. Para que Bootstrap funcione correctamente es necesario agregar las bibliotecas JQuery y Popper.js.

Frameworks o RUBY ON RAILS o CAKEPHP o ANGULARJS.- Un framework es un conjunto de bibliotecas, herramientas y normas a seguir que ayudan a desarrollar aplicaciones. • Están compuestos por varios componentes que interactúan entre sí. Ruby on Rails. Es un framework o marco de creación de aplicaciones de código abierto que funciona con Ruby. CakePHP es un marco de desarrollo (framework) rápido para PHP. • Libre • Código abierto. Es una estructura que sirve de base a los programadores para que éstos puedan crear aplicaciones web. Angular.js es un software reconocido en el mundo del desarrollo web, se trata de un framework de JavaScript. • Basado en código abierto de Google

Sistemas Administradores de Contenidos (CMS) o Genéricos o Especializados.- Los administradores de contenidos son plataformas para administrar el contenido y diseño de un sitio web. Generalmente poseen los siguientes componentes: • Servidor Web • Módulo de negocio • Manejador de bases de datos Pueden ser: Genéricos. Que se pueden desarrollar varias aplicaciones. Especializados. Creados para una aplicación en específico.

Alojamiento para sitios Web.- Se trata del espacio en donde se aloja un sitio web para ser visible en internet. • También se le conoce como web hosting







e-business

***· Sistemas Administradores de Contenidos para e-bussiness (Definición, características, ejemplos).

Definición.- Los sistemas de Gestión de Contenido (CMS Content Management System) es un sistema online que permite poner en marcha una página web fácil y rápido. Se trata de un software que ayuda en la administración de contenidos dinámicos, ejemplo: un blog, un ecommerce o cualquier tipo de página web. Hoy en día gracias a los CMS cualquier usuario sin conocimientos de programación puede administrar a través de una interfaz gráfica todos los aspectos de una página web.

Características.- • Al utilizar un gestor de contenido, el ahorro de tiempo es una ventaja ya que la creación y puesta en marcha es relativamente instantánea. Este tipo de software posee un buen rendimiento, lo cual, garantiza su correcto funcionamiento. • Los desarrolladores analizan el código y comportamiento de estos sistemas permanentemente, lo cual quiere decir, que si existe algún fallo o duda se aplica a tiempo. Utiliza complementos (para cada uno existen plugins que pueden encargarse de diferentes funciones que dependen de la necesidad del usuario). • La seguridad de estos softwares en cuanto a ataques cibernéticos se encuentra a la vanguardia, lo anterior, derivado de la preocupación de las empresas por la seguridad informática.

Ejemplos: WORDPRESS. Sistema sencillo e intuitivo con el que un usuario puede llegar a construir un sitio web complejo. Esta herramienta ordena directamente los contenidos por orden cronológico. JOOMLA. Se trata de una herramienta líder en creación de web ya que pone a disposición del usuario más de 12,000 mil componentes que permiten ampliar las funcionalidades de tu web. DRUPAL. Es un software libre que utiliza código abierto y está escrito en lenguaje PHP. Los contenidos se almacenan en una base de datos y no en el clásico sistema de ficheros del servidor en forma de archivos estáticos. TYPO3. Facilita al usuario una estructura multinivel, un motor de búsquedas, publicación de contenidos y gestión de la auditoría, cuenta con un mecanismo de uso de plantillas para la maquetación de las páginas. CONTAO. Permite ordenar elementos mediante arrastrar y soltar, copiarlos y pegarlos con el portapapeles e incluso editar múltiples registros a la vez. También, es posible ajustar el título y la descripción de cada página, así como configurar si los motores de búsqueda indexarán un sitio o no. MAGENTO. Proporciona flexibilidad al permitir la instalación de extensiones que incrementan la funcionalidad del sitio web. Tiene la capacidad de gestionar varias tiendas desde el mismo panel de administración y también permite la integración con sistemas externos como ERP. PRESTASHOP. Permite la creación de productos, categorías, cupones de descuento, entre otras cosas, de forma fácil y rápida. Orientado a comercios de tamaño medio debido a su facilidad de uso e instalación. Implementado en PHP e incluye Smarty como sistema gestor de plantillas y MySQL.

***· · Integración de soluciones e-bussines con estrategias de negocio CRM ERP.

ERP es una solución que permite a las compañías centralizar e integrar los procesos y captura de información de áreas como finanzas, ventas, compras, distribución, logística, planeación, producción, gestión de proyectos y recursos humanos, de manera que, las actividades que se asocian a aspectos operativos son automatizadas para que las organizaciones operen de manera óptima bajo un sistema estandarizado.

El CRM, es una estrategia de negocios que se apoya en el uso de las TIC que permite mejorar las relaciones de una empresa con sus clientes y prospectos, incrementar la presencia de sus clientes más rentables, aumentar el valor en sus interacciones y reducir costos. Se basa en el conocimiento de los perfiles de los clientes para generar relaciones con ellos, acercarlos y fidelizarlos a la empresa; lo anterior, conlleva acciones de mercadotecnia dirigidas y la creación de servicios al cliente que ayuden a generar una imagen de valor (productos y/o servicios) hacia los clientes.

SCM (Sistema gestor de cadena de suministros), forma parte del CRM, se encarga de planear, organizar y coordinar la cadena de suministros. Se refiere a la forma en que la empresa hace llegar recursos a sus procesos, desde el proveedor de los mismos hasta su transformación proceso a proceso.
